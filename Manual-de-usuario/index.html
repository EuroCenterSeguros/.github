Tareas Programadas

Plesk incluye un gestor de tareas programadas (cron jobs) para automatizar procesos periódicos.

Cómo acceder a las tareas programadas
  1. Accede a Plesk (nuevo o antiguo según corresponda)
  2. En el panel lateral, ve a Herramientas y configuración
  3. Busca "Tareas programadas" o "Scheduled Tasks"
  4. Selecciona el dominio/suscripción donde quieres gestionar las tareas
  5. Puedes crear, editar, activar/desactivar o eliminar tareas desde aquí
Formato de una tarea cron
Ejemplo cron # Cada día a las 8:00 AM 0 8 * * * /usr/bin/php /var/www/vhosts/eurocenterseguros.org.es/httpdocs/script.php # Cada hora 0 * * * * /usr/bin/php /ruta/al/archivo.php # Cada lunes a las 9:00 0 9 * * 1 /usr/bin/php /ruta/al/archivo.php
ℹ️
Plesk también permite configurar tareas desde la interfaz gráfica seleccionando frecuencia (minutos, horas, días) sin necesidad de escribir la sintaxis cron manualmente.

httpdocs y carpeta usuarios

La estructura de directorios del servidor sigue la jerarquía estándar de Plesk. Conocer estas rutas es fundamental para modificar o subir archivos.

Estructura de directorios
Árbol de directorios /var/www/vhosts/ ├── eurocenterseguros.org.es/ │ └── httpdocs/ ← raíz web pública │ ├── usuarios/ ← PHP de gestión de usuarios │ │ ├── buscaSiniestro4.php │ │ ├── pagoseguro.php │ │ └── formu.html │ ├── Form/ │ │ └── formulario.php │ └── ... └── eurocenterseguros.com.es/ └── httpdocs/ └── EnvioCorreos/ └── Controlador/ ← sistema de envío de correos
Cómo navegar en el gestor de archivos de Plesk
  1. Accede a Plesk y ve al dominio correspondiente
  2. Haz clic en "Gestor de archivos" en el panel del dominio
  3. Llegarás a /httpdocs — la raíz pública del dominio
  4. Para acceder a los PHP de usuarios: navega a la carpeta usuarios/
  5. Para los PHP de envío de correos: navega a EnvioCorreos/Controlador/
⚠️
No subas ni modifiques archivos directamente en producción sin hacer una copia de seguridad previa. Usa el modo test para verificar cambios antes de aplicarlos.

El sistema de correo está gestionado mediante PHP en el dominio eurocenterseguros.com.es. Centraliza todos los envíos automáticos de la correduría.

Modo Test / Pruebas

Cada PHP de la empresa dispone de un apartado de test para realizar pruebas de forma segura, enviando los correos únicamente al desarrollador en lugar de a los destinatarios reales.

✅
Regla de oro: Siempre activa el modo test antes de hacer pruebas. Así garantizas que ningún cliente recibe correos de prueba accidentalmente.
Cómo activar el modo test
  1. Localiza el archivo PHP que quieres probar en el servidor
  2. Busca la variable o constante de modo test — habitualmente al inicio del archivo
  3. Cambia el valor a true o 1 según el PHP
  4. Guarda el archivo y realiza la prueba
  5. Importante: el correo se enviará solo al email del desarrollador configurado
  6. Una vez verificado que todo funciona, vuelve a desactivar el modo test (false / 0)
Ejemplo típico de variable de test
PHP // Modo test: true = solo envía al desarrollador // Cambiar a false para envío real a clientes $modo_test = true; $email_desarrollador = "developer@eurocenterseguros.com.es"; if ($modo_test) { $destinatario = $email_desarrollador; } else { $destinatario = $email_cliente; }
🚨
¡No olvides desactivar el modo test! Dejar el modo test activo en producción significa que los clientes no recibirán sus correos. Comprueba siempre el estado antes de dar por terminado cualquier cambio.

El sitio web original de la empresa, alojado en eurocenterseguros.org.es. Se mantiene operativo pero con limitaciones importantes.

🔗 Acceder al WP Antiguo
🚨
ATENCIÓN — WordPress Legacy:
  • Versión de WordPress desactualizada
  • Plugins con actualizaciones pendientes — NO actualizar sin pruebas previas
  • Actualizar plugins o el core puede romper el sitio

¿Qué SÍ puedes hacer?
  • ✅ Editar textos de páginas existentes con cuidado
  • ✅ Gestionar entradas del blog
  • ✅ Consultar configuración y ajustes
  • ✅ Revisar formularios y configuraciones de plugins activos
¿Qué NO debes hacer?
  • 🚫 Actualizar WordPress core
  • 🚫 Actualizar plugins masivamente
  • 🚫 Instalar nuevos plugins sin consultar
  • 🚫 Cambiar la plantilla o tema activo
